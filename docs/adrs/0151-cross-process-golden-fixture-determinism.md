# ADR-0151: Cross-process golden-fixture determinism test with centralized timestamp redaction

## Context

Introduced for cxpak v2.1.0 ("The Polish"). The self-contained SPA dashboard embeds
intelligence output as inline JSON. HashMap iteration order can leak into that JSON, and
because a HashMap's seed is stable within a single process, two `render_spa()` calls in one
process would produce identical output and pass a naive determinism check trivially — masking
nondeterminism that only manifests across processes (e.g. HashMap order persisted into JSON
across separate runs). We needed a determinism check that exercises the cross-process path
and that tolerates the handful of fields (timestamps, version string) that are legitimately
nondeterministic.

## Options considered

- **Option A — Committed cross-process golden fixture diffed against a fresh render:**
  `tests/snapshots/spa_golden.html` is committed; the test diffs it against a fresh
  `render_spa()` of the fixed fixture repo `tests/fixtures/determinism_repo`. `UPDATE_SNAPSHOTS=1`
  regenerates the golden locally and is the only write path; CI leaves the variable unset so any
  drift fails. A missing golden fails with a bootstrap message rather than auto-creating.
  Pros: catches cross-process nondeterminism; snapshot updates are deliberate. Cons: the golden
  churns on legitimate output changes and must be regenerated by hand. Chosen.
- **Option B — Two in-process `render_spa()` calls compared for equality:** Render twice in one
  test and assert equality. Pros: trivial to write. Cons: the stable in-process HashMap seed makes
  it pass trivially and miss exactly the cross-process order leaks this test exists to catch.
  Someone could prefer it for its zero maintenance and no committed artifact, but it does not
  provide the guarantee.

## Decision

Use a committed cross-process golden fixture (`tests/snapshots/spa_golden.html`) diffed against a
fresh render of a fixed fixture repo (`tests/fixtures/determinism_repo`). `UPDATE_SNAPSHOTS=1` is
honored only locally to regenerate the golden; CI runs with it unset so any drift fails. A missing
golden file fails with `run UPDATE_SNAPSHOTS=1 cargo test to bootstrap <path>` rather than
auto-creating. `build_search_index` sorts before returning and ordered collections (BTreeSet for
module dedup) are used wherever serialization determinism matters.

Unavoidable nondeterministic fields — `metadata.generated_at`, `metadata.cxpak_version`,
`ArchitectureSnapshot.timestamp`, `BaselineComparison.baseline_date`, and any future timestamp —
are redacted to fixed placeholders before comparison. Redaction is split across two mechanisms:
`tests/support/redact.rs` centralizes JSON `Value` redaction for the cross-channel
byte-equality tests (`tests/cross_channel_consistency.rs`), while the SPA HTML golden uses a
parallel inline `redact_html()` regex plus hardcoded `[REDACTED]` `RenderMetadata` fields in
`tests/spa_determinism.rs`. The central helper is not used by the SPA determinism test itself.

The SPA determinism test is gated to `target_os = "macos"`, because the rendered snapshot is
platform-dependent (x86_64 Linux re-renders differ on f64/FMA corner cases). Cross-platform byte
parity is explicitly not achieved and is deferred.

Note: the design doc named the drift struct `DriftSnapshot`; the shipped type is
`ArchitectureSnapshot`. Its `timestamp` field redacts correctly regardless.

## Consequences

### Positive
- Detects HashMap-order nondeterminism that only manifests across separate processes.
- Snapshot updates are deliberate (CI cannot regenerate); JSON cross-channel redaction is
  centralized in one helper.

### Negative
- The golden fixture must be regenerated by hand on every intentional output change.
- Redaction is not fully centralized: the SPA HTML golden uses its own inline regex redactor
  rather than `tests/support/redact.rs`.

### Neutral
- Determinism is verified only on macOS; cross-platform byte parity is deferred.
- Repo history confirms ongoing `spa_golden.html` refreshes and a determinism-fixture
  git-isolation fix.

## Revisit if
- A new nondeterministic field appears (add it to the redaction path — currently two paths).
- Cross-platform byte parity becomes a requirement (lift the macOS-only gate).
