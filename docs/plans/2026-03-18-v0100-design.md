# cxpak v0.10.0 Design: 42 Languages + Search + Focus

## Summary

Expand cxpak from 12 to 42 languages, add a regex search MCP tool, and add path-scoped filtering to all MCP tools.

## Languages

### New Languages (30)

All sourced from crates.io. Each is an optional dependency behind a Cargo feature flag, included in `default`. All 30 crates verified to compile against `tree-sitter = "0.25"`.

### `SymbolKind` Extension

Tier 2 languages need new `SymbolKind` variants. Add these to the existing enum in `src/parser/language.rs`:

```rust
pub enum SymbolKind {
    // Existing
    Function, Struct, Enum, Trait, Interface, Class, Method, Constant, TypeAlias,
    // New — Tier 2 structural units
    Selector,    // CSS/SCSS rules
    Mixin,       // SCSS mixins
    Variable,    // SCSS/Makefile/HCL variables
    Heading,     // Markdown headings
    Section,     // Markdown sections, HTML head/body, Dockerfile stages
    Key,         // JSON/YAML/TOML keys
    Table,       // TOML tables
    Block,       // HCL blocks, generic structural blocks
    Target,      // Makefile targets
    Rule,        // Makefile rules
    Element,     // HTML/XML elements
    Message,     // Proto messages
    Service,     // Proto/GraphQL services
    Query,       // GraphQL queries
    Mutation,    // GraphQL mutations
    Type,        // GraphQL types
    Instruction, // Dockerfile instructions
}
```

Tier 2 `extract()` implementations return `Symbol` structs with these kinds. The rest of the pipeline (index, budget, output, relevance) works unchanged — it operates on `Vec<Symbol>` regardless of kind.

### `detect_language` Extension for Extensionless Files

`detect_language()` currently uses `path.extension()` which returns `None` for files like `Dockerfile`, `Makefile`, `GNUmakefile`. Extend it to also check `file_name()`:

```rust
pub fn detect_language(path: &Path) -> Option<String> {
    // First: check by filename (for extensionless files)
    if let Some(name) = path.file_name().and_then(|n| n.to_str()) {
        let lang = match name {
            "Dockerfile" => Some("dockerfile"),
            "Makefile" | "GNUmakefile" => Some("makefile"),
            _ if name.starts_with("Dockerfile.") => Some("dockerfile"),
            _ => None,
        };
        if let Some(l) = lang {
            return Some(l.to_string());
        }
    }

    // Then: check by extension (existing logic + new extensions)
    let ext = path.extension()?.to_string_lossy().to_lowercase();
    let lang = match ext.as_str() {
        // existing ...
        // new ...
        _ => return None,
    };
    Some(lang.to_string())
}
```

Filename match takes priority over extension match.

### Implementation Pattern

Each language follows the existing pattern:

1. `Cargo.toml`: `tree-sitter-{name} = { version = "...", optional = true }`
2. `Cargo.toml` features: `lang-{name} = ["dep:tree-sitter-{name}"]`, added to `default`
3. `src/scanner/mod.rs`: extension mapping in `detect_language()`
4. `src/parser/languages/{name}.rs`: implements `LanguageSupport` trait
5. `src/parser/languages/mod.rs`: conditional `mod` + registration
6. `src/parser/mod.rs`: registration in `LanguageRegistry::new()`

## `cxpak_search` MCP Tool

### Schema

```json
{
  "name": "cxpak_search",
  "description": "Search codebase content with regex patterns. Returns matching lines with surrounding context.",
  "inputSchema": {
    "type": "object",
    "properties": {
      "pattern": {
        "type": "string",
        "description": "Regex pattern to search for"
      },
      "limit": {
        "type": "number",
        "description": "Maximum number of matches to return (default 20)",
        "default": 20
      },
      "focus": {
        "type": "string",
        "description": "Path prefix to scope search (e.g. 'src/api/')"
      },
      "context_lines": {
        "type": "number",
        "description": "Lines of context before and after each match (default 2)",
        "default": 2
      }
    },
    "required": ["pattern"]
  }
}
```

### Response Format

```json
{
  "pattern": "rate_limit",
  "matches": [
    {
      "path": "src/api/middleware.rs",
      "line": 42,
      "content": "pub fn rate_limit(req: &Request) -> Result<(), RateLimitError> {",
      "context_before": ["", "/// Apply rate limiting to incoming requests."],
      "context_after": ["    let key = req.client_ip();", "    let bucket = self.buckets.get(&key);"]
    }
  ],
  "total_matches": 7,
  "files_searched": 142,
  "truncated": false
}
```

### Implementation

- Promote `regex` crate from transitive dependency (via `ignore` crate) to direct dependency.
- Compile pattern with `regex::Regex::new()`. On invalid regex, return clear error message.
- Iterate `index.files`, skip files not matching `focus` prefix. Skip files with empty `content` (binary files, read failures).
- For each file, search `content` line-by-line. On match, capture the line, line number, and `context_lines` lines before/after (clamped to file boundaries).
- Collect up to `limit` matches. Set `truncated: true` if more exist.
- Default case-sensitive. Pattern syntax is Rust `regex` crate syntax. Users can use `(?i)` prefix for case-insensitive.

### HTTP API

Also exposed as `POST /search` with JSON body on the HTTP server. POST chosen over GET because regex patterns contain characters problematic in URL query strings.

## `focus` Param on All MCP Tools

### Schema Addition

Every existing MCP tool gains:

```json
"focus": {
  "type": "string",
  "description": "Path prefix to scope results (e.g. 'src/', 'tests/')"
}
```

### Behavior Per Tool

| Tool | `focus` behavior |
|---|---|
| `cxpak_overview` | `total_files`, `total_tokens`, and `languages` breakdown filtered to only files under prefix |
| `cxpak_trace` | Only trace into files under prefix. If a dependency is outside the prefix, it is listed in an `out_of_scope_deps` array but not traversed. |
| `cxpak_diff` | Only show changes to files under prefix |
| `cxpak_stats` | File count, tokens, languages scoped to prefix |
| `cxpak_context_for_task` | Only score files under prefix |
| `cxpak_pack_context` | Selected files must be under prefix. Dependencies outside prefix are included (they are explicitly needed) but flagged as `"included_as": "out_of_scope_dependency"`. |
| `cxpak_search` | Built-in (see above) |

### Implementation

Single utility function applied as early filter:

```rust
fn matches_focus(path: &str, focus: Option<&str>) -> bool {
    focus.map_or(true, |f| path.starts_with(f))
}
```

`focus` with or without trailing slash both work: `"src/"` and `"src"` match `"src/api/mod.rs"`.

### Compile-Time Impact

Adding 30 tree-sitter grammars increases compile time significantly (each grammar compiles a C parser from scratch). Clean builds may take 3-5 minutes. Incremental builds are unaffected — C grammars are only recompiled when the crate version changes.

Users who want faster builds can use `--no-default-features` and enable only the languages they need. An `all-languages` feature alias is not needed — `default` already serves this purpose.

## Design Decisions

| Decision | Choice | Rationale |
|---|---|---|
| All languages in `default` | Yes | Users compile all by default; can opt out with `--no-default-features` |
| Tier 1 vs Tier 2 split | By language type | Config/markup don't have functions — extract structural units instead |
| New `SymbolKind` variants | 17 new variants | Tier 2 needs them; existing pipeline operates on `Vec<Symbol>` generically |
| `.m` ambiguity | Objective-C unconditionally | Simple, predictable; revisit with heuristics later |
| `.zsh` excluded | Not mapped | `tree-sitter-bash` produces incorrect parses for zsh-specific syntax |
| OCaml single language | One registration, extension-based grammar selection | `.ml` → `LANGUAGE_OCAML`, `.mli` → `LANGUAGE_OCAML_INTERFACE` |
| PHP grammar | `LANGUAGE_PHP` (not `LANGUAGE_PHP_ONLY`) | Full PHP with `<?php` tags |
| XML grammar | `LANGUAGE_XML` (not `LANGUAGE_DTD`) | DTD is niche; XML covers all listed extensions |
| `detect_language` filename check | Filename match before extension match | Handles `Dockerfile`, `Makefile`, `GNUmakefile` |
| Regex crate | Promoted from transitive to direct dependency | Already in dep tree via `ignore`; no new compile cost |
| `focus` as prefix match | `path.starts_with(f)` | Simple, predictable, covers all use cases |
| Search HTTP endpoint | POST (not GET) | Regex patterns have URL-problematic characters |
| Case sensitivity | Default case-sensitive | Matches developer expectations; `(?i)` available |
| `focus` on trace | Out-of-scope deps listed but not traversed | Avoids silent incompleteness; user sees what was skipped |
